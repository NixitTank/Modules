2. What will be the result for these expressions?

            1. 5 > 4
            2. "apple" > "pineapple"
            3. "2" > "12"
            4. undefined == null
            5. undefined === null
            6. null == "\n0\n"
            7. 7. null === +"\n0\n"

1. 5 > 4

This expression checks if 5 is greater than 4.

Result: true.

2. "apple" > "pineapple"

This expression compares the strings "apple" and "pineapple" lexicographically (dictionary order).

Since the first character of "apple" ('a') is less than the first character of "pineapple" ('p'), the result is false.

Result: false.

3. "2" > "12"

This expression compares the strings "2" and "12" lexicographically.

Lexicographically, "2" is greater than "12" because the comparison is done character by character, and '2' is greater than '1'.

Result: true.

4. undefined == null

This expression checks if undefined is equal to null using loose equality (==).

In JavaScript, undefined and null are loosely equal.

Result: true.

5. undefined === null

This expression checks if undefined is strictly equal to null using strict equality (===).

In JavaScript, undefined and null are not strictly equal.

Result: false.

6. null == "\n0\n"

This expression checks if null is loosely equal to the string "\n0\n".

In JavaScript, null is only loosely equal to undefined, not to any other value.

Result: false.

7. 7. null === +"\n0\n"

This expression checks if null is strictly equal to the result of converting the string "\n0\n" to a number.

The string "\n0\n" when converted to a number results in 0.

In JavaScript, null is not strictly equal to 0.

Result: false.
